## 🏗️ 1. Architecture Overview (Microservices)

The project is built on a distributed microservices architecture. It uses **Next.js (App Router)** for the frontend and multiple independent **FastAPI (Python 3.12)** backends.

*   **Database:** PostgreSQL hosted on [Neon.tech](https://neon.tech/).
*   **Storage:** AWS S3 (`michicondrias-storage-1`). All heavy media (images/documents) MUST be uploaded directly from the frontend to S3 using Pre-signed URLs generated by the backend. **Never** send base64 image strings to the backend (AWS API Gateway 6MB limit constraint).
*   **Cloud Hosted:** AWS Lambda + API Gateway (`https://kowly51wia...`).

### Backend Services
All services run locally via `docker-compose` or individually using FastAPI/Uvicorn.
*   `michicondrias_core` (Port 8000): Auth, Users, Security, KYC validation.
*   `michicondrias_adopciones` (Port 8001): Adoption listings and requests.
*   `michicondrias_directorio` (Port 8002): Vet and Clinic directory.
*   `michicondrias_carnet` (Port 8003): Medical records and vaccination logs.
*   `michicondrias_ecommerce` (Port 8004): Shop, Products, Orders.
*   `michicondrias_mascotas` (Port 8005): Permanent Pet profiles & Digital Passport.
*   `michicondrias_perdidas` (Port 8006): Lost pets, Geolocation, Michi-Tracker.

## 🎨 2. Design Aesthetics: "Premium Michi-Style"

The user expects a **stunning, ultra-premium UI/UX**. If a component looks "generic, basic, or like default bootstrap", you have failed.
*   **Core Aesthetic:** Dark mode by default, Deep-blur Glassmorphism (`backdrop-filter: blur(20px)`), Neon Glowing borders (`box-shadow: 0 0 Xpx var(--neon-color)`).
*   **Color Palette (Variables):**
    *   Primary Neon: `var(--neon-green)` / `var(--neon-blue)`
    *   Alert/Danger: `var(--neon-red)` / `var(--neon-orange)` (used for lost pets or destructive actions).
    *   Surfaces: Semi-transparent dark grays (`rgba(20, 20, 25, 0.7)`).
*   **Typography:** Modern Sans-serif (Inter/Outfit). High-contrast text colors (`var(--text-primary)`, `var(--text-secondary)`).
*   **Interactions:** Smooth CSS transitions (`transition: all 0.3s ease`). Hover effects that lift cards and intensify the neon glow.
*   **No Native Alerts:** NEVER use browser native `alert()`, `confirm()`, or `prompt()`. ALWAYS use custom Premium Modals (Glassmorphism + SVG icons).
*   **Icons:** Use high-fidelity SVG paths.

## 🛠️ 3. Frontend Standards (Next.js)

*   **CSS Architecture:** We use **CSS Modules** (`styles.module.css`). Every major page/component should have its own module to prevent global scope pollution.
*   **API Calls:** ALWAYS use the `apiFetch` wrapper located in `src/lib/api.ts`. Do not use naked `fetch()` calls. 
*   **State Management:** We use **React Query** (`@tanstack/react-query`) for data fetching, caching, and optimistic UI updates (especially for high-volume data like pending requests or dashboards).
*   **Services:** API request definitions are abstracted into files like `src/lib/services/adopciones.ts`. Look there first before writing a new `apiFetch` call.
*   **Authentication:** Managed via JWT (`access_token` in `localStorage`). Managed by `src/lib/auth.ts`.
